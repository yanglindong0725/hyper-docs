这是一篇关于文档编写规范的指南。

## 项目文档目录结构

项目文档的目录结构如下：

```md
├── astro.config.mjs
├── src
│   ├── content
│   │   ├── docs
│   │   │   ├── new-project
│   │   │   │   ├── hello-world.md
│   │   │   │   └── ...
│   │   │   ├── guides
│   │   │   │   ├── document-writing-specification.md
│   │   │   │   └── ...
```

## 命名规范

项目文档的文件名和文件夹名必须使用小写字母，单词之间使用`-`连接。例如: `hello-world.md` 或 `new-project`。

## 目录层级规范

项目文档的目录层级必须遵循以下规范：

- 目录：`docs`，用于分类项目, 例如：`docs/new-project`。

## 文档标题规范

项目文档的标题必须遵循以下规范：

- 一级标题编写在 `---` 之间的 `title` 字段中。例如：

```md
---
title: 文档标题
description: 文档描述
---
```

- 二级标题使用 `##` 编写。例如：

```md
## 二级标题

```

- 三级标题使用 `###` 编写。例如：

```md
### 三级标题
```

- 文档中最多只能出现三级标题。
